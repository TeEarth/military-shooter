## 1. เอฟเฟกต์เปิดกาชาให้อลังการขึ้น

ตอนนี้เปิดกาชาแบบเรียบๆ เกินไป ให้ทำ animation sequence จริงจัง (ทำในหน้า `GachaClient.tsx` ด้วย CSS animation/keyframes หรือ Framer Motion ก็ได้):

1. แคปซูล (`shop_gacha_capsule.svg`) เด้ง/หมุนกลางจอ 1-1.5 วิ
2. แคปซูล "แตก" (scale + fade out เร็วๆ พร้อม particle เล็กๆ กระจาย)
3. แสดง `gacha_burst.svg` เป็นพื้นหลังเรืองแสงขยายออกจากจุดกึ่งกลาง (scale 0→1.4 + fade, ประมาณ 0.4-0.6 วิ) พร้อมกัน sprite ไอเทมที่ได้ (helmet/vest/boots ตาม rarity) ค่อยๆ ปรากฏขึ้นตรงกลางแสง (fade-in + scale up แบบ overshoot/bounce)
4. กรอบสีขอบไอเทมตาม rarity (common=เทา, rare=ฟ้า, epic=ม่วง, legendary=ทอง) พร้อม glow เรืองรอบไอเทมแรงขึ้นตาม rarity ที่สูงขึ้น
5. เล่นเสียง `sfx.play("gacha_reveal")` ที่มีอยู่แล้วจังหวะแคปซูลแตก
6. ถ้าเป็น legendary ให้เพิ่มเอฟเฟกต์พิเศษกว่าปกติ (เช่น glow กระพริบ/ขยายรัศมีมากกว่า rarity อื่น) ให้รู้สึกว่าของหายากจริง

## 2. ระบบไอเทมซ้ำ (dupe) + จัดหมวดหมู่ Inventory + อัปเกรดด้วยดาว

**ตอนนี้ระบบ dedup ที่ระบุไว้ใน v3 ยังไม่ได้ implement จริง — ให้ทำจริงตามนี้:**

- ทุกครั้งที่กาชาออกไอเทมที่ผู้เล่นมีอยู่แล้ว (ชิ้นเดียวกัน, slot เดียวกัน, rarity เดียวกัน) → เพิ่ม `PlayerEquipmentLevel.upgradeLevel` ของไอเทมนั้น +1 ทันที (ไม่เพิ่มเป็นชิ้นใหม่ในคลัง) ตามที่ระบุไว้ใน v3 ข้อ 3
- หน้า Inventory: จัดกลุ่มไอเทมเป็นหมวดหมู่ชัดเจน (แท็บหรือ section แยก: อาวุธ / หมวก / เกราะ / รองเท้า) ไม่ปนกันเป็น grid เดียว
- กดที่ไอเทมแต่ละชิ้น (ที่เป็น equipment) ให้เปิด popup/detail แสดง: ค่าสถานะฐานของ rarity นั้น, ระดับอัปเกรดปัจจุบัน, ปุ่ม/ข้อมูลว่าต้องได้ dupe อีกกี่ชิ้นถึงจะอัปเกรดเพิ่ม (ถ้าอยากให้กดอัปเกรดเองแทนอัตโนมัติ ก็ทำปุ่ม "ใช้ของซ้ำที่มี X ชิ้นเพื่ออัปเกรด" แทนก็ได้ — เลือกแบบไหนก็ได้ แต่ต้องมี UI ให้เห็นและกดโต้ตอบได้จริง ไม่ใช่แค่ auto เงียบๆ)
- **แสดงดาว** ใต้/มุมไอคอนไอเทมทุกชิ้นที่ upgradeLevel > 0 โดยจำนวนดาว = `upgradeLevel` (ใช้ `star_upgrade.svg`) ถ้า upgrade เกิน 5 ระดับ ให้แสดงเป็น "★ x N" แทนการเรียงดาวเป็นแถวยาว

## 4. Layout หน้า Inventory ใหม่ (ปรับจาก v2 ข้อ 5)

แก้ layout ให้เป็นแบบนี้แทน:

- ตัวละคร + ช่อง equip 4 มุม (ตามที่ทำไว้แล้วใน v2) อยู่ฝั่งซ้ายของจอ
- **เพิ่มแผงสถานะ (stat panel) ทางด้านขวาของตัวละคร** ชิดซ้ายของแผงตัวเอง (ไม่ต้องเต็มความกว้างจอ เว้น margin ให้ดูโปร่ง) แสดง:
  - สถานะรวมทั้งหมดของผู้เล่น (HP, ATK, ความเร็ว, ความแม่นยำ, เกราะ%, คริติคอล%, ดาเมจคริติคอล%, ความเร็วรีโหลด, ความเร็วยิง, กระสุนรายวัน, **โล่รวม**) แบบ "ฐาน + โบนัส% = รวม" ตามที่ระบุไว้ใน v3 ข้อ 4.13
  - แยก section ย่อยด้านล่างสถานะรวม: **"โบนัสจากอุปกรณ์ที่ใส่อยู่"** แสดงทีละชิ้น (หมวก/เกราะ/รองเท้า) ว่าชิ้นนั้นให้ค่าอะไรบ้าง เช่น:
    ```
    หมวก (Epic, ★★): HP +16%, ATK +16%, CritDMG +20%, Shield +300
    เกราะ (Rare): HP +10%, ATK +10%, Shield +200
    รองเท้า (Common): HP +3%, Shield +50
    ```
  - จัดวางเป็น card แยกแต่ละหมวด เว้นระยะห่างชัดเจน (ใช้ grid/flex + gap แทนการเรียงชิดกัน) ให้อ่านง่าย ไม่ใช่ข้อความยาวติดกัน

## 8. Hitbox สิ่งกีดขวางใหญ่เกินภาพ

ตอนนี้ collision box ของ cover object ใหญ่กว่ารูปที่เห็นจริง ทำให้ดูเหมือนมีช่องว่างเดินไม่ได้ทั้งที่ภาพไม่ได้บังตรงนั้น — แก้ `CoverObject.ts` ให้ physics body ขนาดตรงกับพื้นที่จริงของภาพ (ใช้ `body.setSize()`/`setOffset()` ให้ตรงกับ bounding box จริงของแต่ละ sprite ในข้อ 8 ของ v5 ทั้ง 6 แบบ ไม่ใช้ default bounding box ของทั้ง texture เต็มขนาด) — ยืนยันว่ากระสุนยังชนแล้วหายเหมือนเดิม (ห้ามทะลุ) แค่ปรับขนาด hitbox ให้พอดีภาพเท่านั้น

## 9. Reset test account (แก้จาก v5 ข้อ 2) — ไม่ auto-grant ปืน/ตัวละคร + แจ้ง credential ทุกครั้ง

แก้ `scripts/reset-test-account.ts`:

- ตั้ง coin, diamond, ticket เป็นค่าสูงมาก (unlimited) เหมือนเดิมจาก v5
- **ผู้ใช้จะลบ row เดิมของบัญชีนี้ใน Google Sheet `Players` เองก่อนรันสคริปต์ทุกครั้ง** ดังนั้นสคริปต์ต้องรองรับกรณี "ไม่เจอ player เดิม" โดยสร้างบัญชีใหม่ให้เลย (ใช้ flow เดียวกับ guest/register ที่มีอยู่ใน `/api/auth/guest` หรือ `/api/auth/register`)
- หลังสร้าง/รีเซ็ตเสร็จ **ให้ print ออกทาง console เสมอทุกครั้งที่รัน**: playerId ใหม่, email ที่ใช้, วิธี login (เช่น ลิงก์/username/password ชั่วคราวถ้าใช้ระบบ guest token) เพื่อให้ผู้ใช้เอาไป login ทดสอบได้ทันที

## 10. บั๊กตัวละครหาย — ยังไม่หาย โดยเฉพาะตอนเล่น Azzure

ให้ debug เจาะจงกรณี Azzure เพราะ sprite ของตัวนี้ (`azzure_colonel.svg`) มีองค์ประกอบพิเศษต่างจากตัวอื่น (glow ring รัศมีใหญ่กว่า circle ตัว, เครื่องหมายเหลี่ยมยื่นออกนอกวง) ที่อาจทำให้:
- ขนาด texture ที่โหลดจริงไม่ตรงกับ `body.setCircle()`/hitbox ที่ตั้งไว้ (physics body อาจใหญ่/เล็กกว่าที่ควร ทำให้หลุด bounds หรือ z-depth ผิด)
- ตรวจว่า key sprite `azzure_colonel` ถูก preload สำเร็จจริงก่อนใช้ (เช็ค error/console warning ตอนโหลด SVG นี้เทียบกับตัวอื่น)
- ตรวจ depth/z-index ของ player container ไม่ให้ background หรือ cover object บังจนดูเหมือนหายไป
- ให้ log ตำแหน่ง/สถานะ visible ของ player sprite ทุกเฟรมชั่วคราว (debug mode) ตอนเล่น Azzure โดยเฉพาะ เพื่อยืนยันว่า sprite ยังอยู่แต่มองไม่เห็น หรือ object ถูก destroy ไปจริงๆ แล้วแก้ตามสาเหตุที่เจอจริง (ห้ามแก้เดา)

## 13. ความเร็วกระสุนต้องเท่ากันทุกกระบอก ยกเว้น Sniper และ Grenade Launcher

แก้ให้มี `BASE_BULLET_SPEED` ค่าเดียวใน config (เช่น `config/game.ts`) ที่ทุกอาวุธใช้ร่วมกันเป็นค่าเริ่มต้น ยกเว้น:
- **Sniper**: ความเร็วกระสุน = `BASE_BULLET_SPEED x 2`
- **Grenade Launcher**: ความเร็วกระสุน = `BASE_BULLET_SPEED / 2`
- อาวุธอื่นทั้งหมด (pistol, double_pistol, m16a1, m16a4, shotgun, ak47, gatling, rocket_launcher, rasor_gun) ใช้ `BASE_BULLET_SPEED` เท่ากันหมด ไม่มีความเร็วเฉพาะตัวอีกต่อไป — ลบค่า speed เฉพาะอาวุธเดิมที่อาจตั้งไว้ต่างกันออก

## 14. Grenade Launcher: ยิงไปตกที่ตำแหน่งคลิก + ข้ามสิ่งกีดขวางได้จริง

ตอนนี้ยังยิงเป็นเส้นตรงแบบกระสุนปกติและชนสิ่งกีดขวางเหมือนอาวุธอื่น ต้องแก้ mechanic ใหม่ทั้งหมดสำหรับ Grenade Launcher โดยเฉพาะ:

- เปลี่ยนเป็น **click-to-target**: ผู้เล่นคลิก/แตะตำแหน่งบนแมพ ลูกระเบิดจะยิงเป็น arc (โค้งพาราโบลา หรือจะทำเป็นเส้นตรงแต่ไม่ชน collision ก็ได้ ขอแค่ผลลัพธ์คือไปตกที่จุดที่คลิกจริง) แล้วระเบิดสร้างดาเมจเป็นวงที่ตำแหน่งนั้น (ตามที่ระบุไว้ใน v3 ข้อ 3 เรื่อง AoE)
- ระหว่างที่ลูกระเบิดกำลังบิน **ต้องไม่ชน/ไม่ถูก block โดย cover object ใดๆ** (ปิด collision กับ `CoverObject` เฉพาะ projectile ประเภทนี้ หรือทำ layer แยกที่ grenade ไม่ physics-collide ด้วย) เพื่อให้ยิงข้ามกำแพง/ต้นไม้/บ้านได้จริงตามสเปกเดิม
- ใช้ `bullet_grenade.svg` (สร้างไว้แล้ว) เป็น sprite ของลูกระเบิดระหว่างบิน
