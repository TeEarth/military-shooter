# Military Shooter 2D — Master Change Request v7 (ต่อยอดจาก v2-v6)

v2-v6 ยังใช้ได้ทั้งหมด ไฟล์นี้แก้/เพิ่มเฉพาะเรื่องด้านล่าง

## 1. Reset test account ต้องพิมพ์ user/password ออกมาให้เห็นชัดเจนทุกครั้ง (แก้จาก v6 ข้อ 9)

ผู้ใช้ไม่รู้รหัสผ่านที่ระบบสร้างให้ — แก้ `scripts/reset-test-account.ts`:
- ถ้าระบบ login เป็นแบบ email+password (NextAuth credentials): ให้ตั้งรหัสผ่านคงที่ที่รู้ล่วงหน้าเสมอ เช่น `Test1234!` (hash แล้วเขียนลง `Players` ทุกครั้งที่รัน ทับของเดิม) แล้ว **print ออก console ท้ายสคริปต์แบบเห็นชัดเจน** เช่น:
  ```
  ✅ Test account ready
  Email:    earth.noppanut@gmail.com
  Password: Test1234!
  Player ID: xxxxx
  ```
- ถ้าระบบใช้ guest token/login แบบอื่น ให้ print ข้อมูลที่จำเป็นสำหรับ login จริงแทน (เช่น guest token หรือ magic link) — ต้องพิมพ์ให้ครบพอที่ผู้ใช้ล็อกอินเข้าเล่นได้ทันทีโดยไม่ต้องเดา
- ห้ามพิมพ์แค่ email เฉยๆ โดยไม่มี password/วิธี login แนบมาด้วยอีกต่อไป

## 2. บั๊กใหญ่: ฆ่าศัตรูแล้วไม่ได้เงินจริง / ผ่านด่านแล้วไม่ได้รางวัลจริง

นี่คือบั๊กที่กระทบ core economy ทั้งระบบ ต้องหาสาเหตุจริงแล้วแก้ (ห้ามเดา) ให้ตรวจตามลำดับนี้:

1. `/api/game/complete` — เช็คว่า endpoint นี้ถูกเรียกจริงตอนจบด่าน (ทั้งกรณีฆ่าศัตรูครบ/ผ่านด่าน) และ payload ที่ส่งไป (kills, coin ที่ควรได้ตามอัตรารางวัลต่อประเภทศัตรูจาก v2 ข้อ 9) ถูกคำนวณถูกต้องจากฝั่ง client (`GameScene.ts`/`onGameEnd` callback ใน `GameClient.tsx`) ก่อนส่งไป backend
2. ฝั่ง backend: เช็คว่า `/api/game/complete` เขียนค่า coin ใหม่ลง sheet `Players` จริง (มี bug บ่อยเวลาใช้ Google Sheets API แบบ read-then-write ที่ race condition หรืออ่านค่าเก่าตอนคำนวณ ให้ตรวจว่า append/update ใช้ค่าที่อ่านล่าสุดจริง ไม่ใช่ cache เก่าจาก `cache.ts` ที่ TTL 60 วิ)
3. เช็คว่าเหรียญ per-kill ระหว่างเล่น (ถ้ามีระบบให้เงินทันทีตอนฆ่าแต่ละตัวแบบ real-time ไม่ใช่รวมตอนจบด่าน) มีการเรียก API บันทึกจริงหรือแค่ทำ UI popup (`coin_pop.svg` จาก v6) แสดงเฉยๆ โดยไม่ได้บวกเข้าฐานข้อมูลจริง — ถ้าเป็นแบบนี้คือ root cause ให้แก้ให้บันทึกจริงทุกครั้ง (batch ส่งตอนจบด่านก็ได้ ไม่จำเป็นต้องยิง API ทุกครั้งที่ฆ่า แต่ต้องส่งค่ารวมที่ตรงกับที่ UI แสดงจริง)
4. เขียน test เคส manual: ฆ่าศัตรู 3 ตัว (ผสมชนิด) แล้วเช็คใน Google Sheet `Players` ว่ายอด coin เพิ่มตรงตามสูตรจริงหลังจบด่าน ถ้าไม่ตรงให้ debug ต่อจนกว่าจะตรง — ปัญหานี้สำคัญที่สุดในรอบนี้

## 3. เติมกระสุนระหว่างเล่นในด่านได้ (ปรับตัวเลขจาก v3 ข้อ 4)

แก้ตัวเลขระบบเติมกระสุนรายวัน (จาก v3: ดูโฆษณา +10%, จ่ายเพชร 30 = +100%) เป็นค่าใหม่นี้แทน:

- **ดูวิดีโอโฆษณา** ระหว่างเล่นในด่าน (ไม่ต้องออกจากเกม) → เพิ่มกระสุนอาวุธนั้น **+5%** ของโควตารายวัน (ลดจากเดิม 10%)
- **จ่าย 40 เพชร** (เพิ่มจากเดิม 30) → เติมกระสุนอาวุธนั้นเต็ม **100%** ของโควตารายวันทันที
- ต้องกดใช้งานได้จาก HUD ระหว่างเล่นเกมจริง (ไม่ใช่แค่หน้า pause menu) เช่น ปุ่มเล็กๆ ข้างแถบกระสุนใน `HUDScene.ts` เปิด popup เลือก "ดูโฆษณา" หรือ "จ่าย 40 เพชร" ได้ทันทีโดยไม่ต้องออกจากด่าน (เกม pause ชั่วคราวระหว่างดูโฆษณา/ยืนยันจ่ายเพชร)

## 4. อาวุธเริ่มต้น (Pistol) ต้องอยู่ในสถานะ equipped ตั้งแต่แรก

แก้ตอนสร้างผู้เล่นใหม่ (`/api/auth/register`, `/api/auth/guest`, และ script reset): เมื่อ grant pistol ให้ผู้เล่นใหม่ ต้องตั้งค่า `equipped = true` ใน `PlayerWeapon` ทันที (ไม่ต้องรอผู้เล่นกดเลือกเองในหน้าอาวุธ) เพื่อให้เข้าเล่นด่านแรกได้เลยโดยมีอาวุธติดตัวอยู่แล้ว

## 5. ตกแต่งหน้า Character ให้มีรูปภาพ ไม่ใช่แค่ข้อความ

หน้า character/weapon select (`CharacterClient.tsx`) ตอนนี้แสดงแค่ข้อความสถานะ ให้เพิ่ม:
- แสดงรูปตัวละครขนาดใหญ่ (ใช้ sprite ที่มีอยู่แล้ว `bob_private.svg`, `jackson_sergeant.svg`, `ryzor_lieutenant.svg`, `mina_captain.svg`, `azzure_colonel.svg`) เป็นภาพหลักตรงกลาง/ซ้ายของการ์ดตัวละครที่เลือกอยู่
- พื้นหลังการ์ดไล่เฉดสีตามธีมยศ/ตัวละคร (เช่น Azzure ใช้โทนทอง, Ryzor โทนน้ำเงิน ตามสีที่ออกแบบไว้ในตัว sprite เอง)
- เพิ่ม hover/selected state (border เรืองแสงหรือ scale ขึ้นเล็กน้อยตอนเลือก) ให้ดูมีชีวิตชีวาเข้ากับธีมของเกม ไม่ใช่ card สีพื้นเรียบๆ

## 6. Hydration mismatch error ที่หน้า Login

Error ที่ส่งมา (จาก `LoginForm.tsx`) เกิดจาก attribute `fdprocessedid` ที่ปุ่ม/inputs — **นี่คือ browser extension เสริม (พวก autofill/form-filler เช่น LastPass, Dashlane, หรือส่วนเสริมกรอกฟอร์มอัตโนมัติ) แทรก attribute นี้เข้าไปใน DOM ก่อนที่ React จะ hydrate เสร็จ ไม่ใช่บั๊กจากโค้ดเราโดยตรง** เป็น false-positive ที่พบได้บ่อยเวลามี extension พวกนี้ติดตั้งอยู่ในเบราว์เซอร์ที่ทดสอบ

แนะนำให้ Claude Code ทำ 2 อย่างเพื่อลด noise (ไม่ได้แก้ที่ต้นเหตุเพราะต้นเหตุมาจาก extension ของผู้ใช้เอง ไม่ใช่โค้ด):
1. เพิ่ม `suppressHydrationWarning` ที่ elements ที่เป็น interactive input/button ในหน้า `LoginForm.tsx` เพื่อไม่ให้ React ฟ้อง warning นี้ซ้ำอีกในอนาคต (ปลอดภัย เพราะ attribute ที่ extension เติมไม่กระทบ logic ของแอปเรา)
2. แจ้งผู้ใช้ (ในคอมเมนต์โค้ดหรือ README dev notes) ว่าถ้าเทสแล้วเจอ warning แบบนี้อีกที่หน้าอื่น ให้ลองปิด browser extension แล้วโหลดใหม่ก่อนเพื่อยืนยันว่าไม่ใช่บั๊กจริงจากแอป

## ลำดับความสำคัญที่แนะนำสำหรับรอบนี้

1. บั๊กเงินไม่เพิ่มตอนฆ่าศัตรู/ผ่านด่าน (ข้อ 2) — กระทบ core loop รุนแรงที่สุด
2. Reset script โชว์ user/password ชัดเจน (ข้อ 1) — จำเป็นต่อการทดสอบข้อ 2 ต่อ
3. Equip อาวุธเริ่มต้นอัตโนมัติ (ข้อ 4)
4. เติมกระสุนกลางเกม + ปรับตัวเลข (ข้อ 3)
5. ตกแต่งหน้า character (ข้อ 5)
6. Hydration warning suppress (ข้อ 6) — ความสำคัญต่ำสุด ไม่กระทบ gameplay จริง
