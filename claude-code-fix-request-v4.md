# Military Shooter 2D — Master Change Request v4 (ต่อยอดจาก v3)

ไฟล์นี้เป็นส่วนเพิ่มเติมจาก `claude-code-fix-request-v3.md` (v3 ยังใช้ได้ทั้งหมด ไฟล์นี้เพิ่มเรื่อง: เศรษฐกิจเริ่มต้นที่ 0, เควส, การแลกเงิน, ด่านบอส, และแท็บ Income ใหม่)

---

## 1. เริ่มเกมทุกอย่างเป็น 0 (ของหายากทั้งหมด)

แก้ `config/economy.ts`:
```ts
startingCoin: 0,
startingDiamond: 0,
startingTicket: 0,
startingAmmo: 0, // หรือใช้ dailyAmmo ของ pistol ตามปกติในวันแรก
```
ผู้เล่นใหม่มีแค่ตัวละคร Bob + อาวุธ Pistol ฟรีเท่านั้น ทุกอย่างอื่น (ตัวละคร, อาวุธ, อุปกรณ์, เพชร, เหรียญ, ticket) ต้องหาเอง — ปรับราคา/อัตราดรอปทั้งระบบให้สอดคล้องกับแนวคิด "ของหายาก" นี้ (ไม่ต้องเปลี่ยนตัวเลขราคาที่กำหนดไว้แล้วใน v2/v3 แค่ต้องไม่มี starting currency แจกฟรีตอนเริ่ม)

---

## 2. เควสรายวัน + เควสส่วนตัว (ให้เพชร) — ระบบใหม่

ใช้ sheet `Mission`/`PlayerMission` ที่ตั้งไว้แล้วใน v2 มาต่อยอด:

- **เควสรายวัน (reset ทุกวัน):** "ฆ่าข้าศึก 10 ตัว" → รางวัลเพชร 5 เม็ด (นับสะสมทุกโหมด ทั้งด่านทั่วไปและด่านฟาร์ม)
- **เควสส่วนตัว (progression milestone, ได้ครั้งเดียวต่อ milestone ไม่ reset):** ผ่านด่านทั่วไปถึงจำนวนที่กำหนด → เพชร ตามนี้:
  - ผ่านด่าน 5 → เพชร 10
  - ผ่านด่าน 10 → เพชร 10
  - ผ่านด่าน 15 → เพชร 10
  - ผ่านด่าน 20 → เพชร 10
  - จากนั้น milestone ใหม่ทุกๆ 5 ด่านถัดไป (25, 30, 35, ...) ให้เพชร 10 ไปเรื่อยๆ ไม่มีสิ้นสุด (generate milestone ด้วยสูตร ไม่ต้อง hardcode ทีละแถวใน sheet)
- Implementation: เพิ่ม field `killCountToday` (reset รายวัน) และ `stagesCleared` ใน `Players`/progress sheet, เช็ค milestone อัตโนมัติหลังผ่านด่านแต่ละครั้งใน `/api/game/complete`

## 3. ระดับความยากของด่านทั่วไป (stage difficulty scaling + reuse map)

- ทุกๆ 10 ด่าน ศัตรูในด่านทั่วไปทั้งหมด (HP, ดาเมจ, ทุกสถานะ) **+10%** สะสม (เช่น ด่าน 11-20 แข็งแกร่งกว่า 1-10 อีก 10%, ด่าน 21-30 อีก 10% ต่อจากนั้น, compounding)
- แมพจะถูก "วนใช้ซ้ำ" ทุกๆ 10 ด่าน: เช่น ด่าน 11 ใช้ผังแมพเดียวกับด่าน 1 (แค่ยากขึ้น+ศัตรูเยอะขึ้น), ด่าน 12 = ผังด่าน 2, ... เพื่อประหยัดงานออกแบบแมพ (ผู้ใช้จะออกแบบ 10 แมพหลักเองทีหลัง โค้ดแค่ต้องรองรับ mapping `stageNumber % 10` → แมพเทมเพลต)
- จำนวนศัตรูต่อด่านเพิ่มขึ้นทุกรอบ 10 ด่านด้วย (เช่น +1 หรือ +2 ตัวต่อรอบ กำหนดสูตรใน `StageEnemy`/logic ให้ปรับได้ง่ายจาก sheet)
- **ย้ำ:** กติกา "ผ่านด่านแล้วเล่นซ้ำไม่ได้" จาก v2 ยังใช้อยู่เหมือนเดิม อันนี้แค่อธิบายว่าด่านที่ 11, 21, 31, ... เอาผังแมพเก่ามาใช้ซ้ำ ไม่ได้แปลว่ากลับไปเล่นด่าน 1 ซ้ำได้

## 4. ด่านบอส (Bonus map ทุก 10 ด่านหลัก)

- ทุกๆ ผ่านด่านทั่วไปครบ 10 ด่าน (หลังด่าน 10, 20, 30, ...) จะปลดล็อกแมพบอส 1 ด่านให้เล่นก่อนไปด่านถัดไป (นับเป็น bonus stage แยกจาก story stage ปกติ)
- บอส stat เริ่มต้น: HP 2000, ใช้อาวุธแบบ Rocket Launcher ยิงจรวด 5 ลูก (โดนแม้แค่ลูกเดียว = ตายทันที ไม่คิดดาเมจปกติ ให้เป็น instant-kill hit เฉพาะบอส)
- ทุกครั้งที่เจอบอสรอบถัดไป (ครั้งที่ 2, 3, 4...) บอสแข็งแกร่งขึ้น **+10%** จากครั้งก่อนหน้า (compounding เหมือนกัน)
- ฆ่าบอสสำเร็จ: ดรอป "แบงค์เขียว" 1 ใบเสมอ (ดูข้อ 5)
- สร้าง sheet `BossStage` (bossId, hp, weaponId, rocketCount, growthPercent, occursEveryNStages) และ track `bossEncounterCount` ต่อผู้เล่นสำหรับคำนวณ scaling

## 5. แท็บใหม่ "Income" (ย้ายระบบเติมเงินออกจาก Setting)

- ย้าย UI แลกเงินจริง (TrueMoney/เติม ticket) ออกจากหน้า Setting ทั้งหมด ไปสร้างเป็นแท็บเมนูใหม่ชื่อ **Income** (เพิ่มเข้าไปในเมนูหลัก คู่กับ Gacha/Mission/Leaderboard/Mailbox/Setting ที่มีอยู่แล้ว)
- ใน Income มี 2 ส่วน:
  1. **เติม ticket ด้วยเงินจริง** (ย้ายมาจาก Setting เดิม) ตามตาราง:

     | จ่าย (บาท) | ได้รับ (ticket) |
     |---|---|
     | 49 | 50 |
     | 99 | 110 |
     | 199 | 235 |
     | 499 | 600 |
     | 999 | 1,250 |
     | 3,999 | 5,500 |

  2. **แบงค์เขียว (green banknote)** — สกุลเงินใหม่ที่เป็นตัวแทนเงินจริงที่ผู้เล่น "ได้รับ" จากการเล่น (คนละทิศทางกับข้อ 1 ที่จ่ายเงินเข้า):
     - ได้จากการฆ่าบอสสำเร็จ: บอสละ 1 ใบ (ข้อ 4)
     - ได้จากการทำเควสส่วนตัวสำเร็จ: ทุกๆ 5 ครั้งที่ทำเควสส่วนตัวสำเร็จ (นับสะสม) ได้ 5 ใบ
     - อัตราแลก: **1 แบงค์เขียว = 1 บาท**
     - หน้า Income แสดงยอดแบงค์เขียวสะสม + ปุ่ม "ขอถอนเงิน" (withdrawal request)

  **⚠️ ข้อควรระวังสำคัญ (แจ้ง Claude Code และผู้ใช้ให้ทราบ):** ฟีเจอร์ "แลกแบงค์เขียวเป็นเงินจริงถอนออกได้" คือระบบจ่ายเงินจริงออกจากเกม (cash-out) ซึ่ง:
  - ต้องต่อกับผู้ให้บริการชำระเงิน/โอนเงินจริง (payment provider) ที่รองรับการถอนเงิน ไม่สามารถทำเป็นแค่ field ตัวเลขในชีทแล้วจบ — Claude Code ควร implement แค่ "ระบบสะสมยอด + คำขอถอนเงิน (withdrawal request)" ที่บันทึกคำขอไว้ให้แอดมิน (คุณ) ไปดำเนินการโอนเงินเองด้วยมือก่อนในเฟสแรก (manual payout) แทนการต่อ payment API อัตโนมัติทันที เพื่อความปลอดภัยและลดความเสี่ยงเรื่องกฎหมาย
  - การรวมกาชา (สุ่มของ) เข้ากับระบบถอนเงินจริงได้ อาจเข้าข่ายกฎหมายเกี่ยวกับการพนัน/ของรางวัลในประเทศไทย แนะนำให้ปรึกษาผู้เชี่ยวชาญด้านกฎหมายก่อนเปิดให้ผู้เล่นจริงใช้งานฟีเจอร์นี้ (อันนี้เป็นแค่ข้อสังเกต ไม่ได้บล็อกการเขียนโค้ด แต่ควรรู้ไว้ก่อน launch จริง)

## 6. ระบบแลกเปลี่ยนทรัพยากร (แท็บ "Shop" ใหม่ แยกจาก Gacha)

Gacha (v3) ใช้สุ่มอุปกรณ์อย่างเดียว ส่วนการ "แลกเงิน" (currency exchange ตรงๆ ไม่สุ่ม) ให้แยกเป็นเมนูใหม่ชื่อ **Shop** (คนละอันกับ Gacha) มี 2 ตาราง:

**แลกเพชร → เหรียญ**

| จ่ายเพชร | ได้เหรียญ |
|---|---|
| 50 | 200 |
| 100 | 410 |
| 250 | 1,100 |
| 1,000 | 5,000 |

**แลก ticket → เพชร**

| จ่าย ticket | ได้เพชร |
|---|---|
| 50 | 200 |
| 100 | 410 |
| 250 | 1,100 |
| 1,000 | 5,000 |

Implementation: sheet `CurrencyExchangeConfig` (fromCurrency, toCurrency, fromAmount, toAmount), API `/api/shop/exchange` หักสกุลต้นทาง เพิ่มสกุลปลายทาง แบบ atomic (เช็คยอดพอก่อนหักเสมอ)

เมนูหลังสรุปตอนนี้ (แก้จาก v2 ข้อ 10): **Gacha, Shop (แลกเงิน), Mission, Leaderboard, Mail Box, Income, Setting** (Setting เหลือแค่ตั้งค่าทั่วไป เสียง/ภาษา/บัญชี ไม่มีเรื่องเงินแล้ว)
