## 1. Sound Effect — สังเคราะห์เสียงด้วยโค้ด (ไม่ใช้ไฟล์เสียง)

สร้างไฟล์ `src/lib/sfx.ts` ไว้ให้แล้ว (พร้อมใช้ทันที ไม่ต้องรอไฟล์เสียงจากที่ไหน) เป็นระบบสังเคราะห์เสียงด้วย Web Audio API ล้วนๆ (oscillator + noise buffer) ไม่มี asset ไฟล์เสียงให้จัดการ, import ใช้ผ่าน `import { sfx } from "@/lib/sfx"` แล้วเรียก `sfx.play("shoot_pistol")` เป็นต้น

ให้ Claude Code เดินสายเรียกใช้ตามคอมเมนต์ `WIRING NOTES` ท้ายไฟล์ `sfx.ts`:
- ยิงปืนแต่ละประเภท (แยกเสียงตาม fireMode: pistol/rifle/shotgun/sniper/heavy-auto)
- ระเบิด (rocket/grenade ตอนกระทบเป้าหมาย)
- รีโหลด, โดนศัตรู (hit), โดนศัตรูตี (hurt), เดิน (footstep, throttle ~2-3 ครั้ง/วิ), เก็บเหรียญ/ไอเทม, กดปุ่ม UI ทุกหน้า, เปิดกาชา (reveal), ชนะ/แพ้ด่าน (GameOverScene)
- เพิ่ม toggle mute/volume ในหน้า Setting เชื่อมกับ `sfx.setMuted()` / `sfx.setVolume()`

ถ้าภายหลังอยากได้เสียงจริงคุณภาพสูงกว่านี้ (เสียงปืนจริง, เพลงประกอบ) ต้องใช้ไฟล์เสียงจากภายนอก (เช่น freesound.org แบบ royalty-free) เพราะ sandbox นี้สร้างไฟล์เสียงจริงไม่ได้ — โค้ดสังเคราะห์นี้เป็นทางออกชั่วคราวที่ใช้งานได้จริงและฟรี 100%

## 2. รีเซ็ตบัญชีทดสอบ แบบทำซ้ำได้ทุกครั้ง (แก้จาก v2 ข้อ 0)

เดิม v2 ให้ทำสคริปต์รีเซ็ตครั้งเดียว ตอนนี้เปลี่ยนเป็น **เครื่องมือทดสอบที่รันซ้ำได้ตลอด**:

- สร้าง `scripts/reset-test-account.ts` รันด้วย `npx tsx scripts/reset-test-account.ts --email=earth.noppanut@gmail.com` (หรือเพิ่ม `npm run test:reset` ใน package.json ที่รับ email จาก env/arg)
- สิ่งที่สคริปต์ทำทุกครั้งที่รัน:
  1. รีเซ็ต progress ด่าน: `stagesCleared = 0`, `farmStageMaxWave = 0`, `bossEncounterCount = 0`, ล้าง `PlayerStageProgress` ทั้งหมดของ playerId นี้ (กลับไปเริ่มด่าน 1)
  2. **ไม่ปลดล็อกตัวละคร/อาวุธ/อุปกรณ์ทั้งหมดให้อัตโนมัติ** — คงไว้แค่ Bob + Pistol เริ่มต้นเหมือนผู้เล่นใหม่จริง (เพื่อให้ทดสอบการซื้อ/กาชาจริง)
  3. ตั้งค่า **coin, diamond, ticket ให้เป็นตัวเลขจำนวนมากพอที่ถือว่า "ไม่จำกัด"** สำหรับทดสอบ (เช่น 999,999,999 ทั้ง 3 สกุลเงิน) เพื่อให้ทดลองซื้อของ/กาชา/แลกเงินได้ทุกอย่างจริงโดยไม่ติดขัดเรื่องเงินไม่พอ
  4. รีเซ็ต quest ทั้งหมด (daily + personal) กลับเป็นยังไม่ทำ
  5. ล้าง `PlayerWeaponAmmo` กลับเต็มโควตารายวันของทุกอาวุธที่ครอบครอง
- ใช้สคริปต์นี้รันใหม่ได้ทุกครั้งก่อนเริ่มทดสอบรอบใหม่ (ไม่ใช่ one-off อีกต่อไป)

## 3. แก้ไข Inventory ลากวางยังใช้งานไม่ได้จริง (แก้จาก v2 ข้อ 5)

ที่ทำไปตาม v2 ยังไม่ทำงานจริง — ให้ตรวจสอบและแก้ implementation จริง ไม่ใช่แค่ใส่ `draggable` attribute เฉยๆ:
- ใช้ไลบรารี `@dnd-kit/core` (`npm install @dnd-kit/core`) แทนการเขียน native HTML5 drag events เอง (native DnD API มีปัญหาเรื่อง touch/mobile และ event propagation บ่อย)
- ต้องมี `DndContext` ครอบทั้งหน้า inventory, แถบไอเทมด้านล่างเป็น `useDraggable` แต่ละไอเทม, ช่อง 4 มุมเป็น `useDroppable` ที่ validate ประเภทไอเทม (slot type ต้องตรงกับ droppable id เช่น ลากอาวุธไปวางช่องหมวกต้อง reject แล้วเด้งกลับที่เดิม พร้อม toast แจ้งเตือน)
- หลัง drop สำเร็จ: เรียก API equip ทันที (ตามที่ออกแบบไว้ใน v2 ข้อ 4.1/5), อัปเดต UI แบบ optimistic แล้ว sync กับ response จริง
- ทดสอบบนทั้ง desktop (mouse) และมือถือ (touch) เพราะ `@dnd-kit` รองรับทั้งคู่โดยธรรมชาติ

## 4. แพ้ด่านแล้วต้องเริ่มใหม่ทั้งหมด → แก้ให้กด "เล่นด่านนี้ซ้ำ" ได้ทันที

บั๊ก/ดีไซน์ผิด: ตอนนี้เมื่อผู้เล่นตาย (แพ้) ระบบบังคับกลับไปเริ่มตั้งแต่ด่านแรกใหม่ ซึ่งผิด ให้แก้เป็น:

- เมื่อแพ้ด่านทั่วไป (ไม่ว่าด่านที่เท่าไหร่): ขึ้นหน้าจอ "GAME OVER" พร้อมปุ่ม **"ลองด่านนี้อีกครั้ง (Retry)"** และ "กลับหน้าหลัก" — กด Retry แล้วโหลดด่านเดิมใหม่ทันที **ไม่รีเซ็ต progress ด่านอื่นที่ผ่านไปแล้ว** (stagesCleared ของด่านก่อนหน้ายังคงอยู่)
- กติกา "ผ่านด่านแล้วเล่นซ้ำไม่ได้" (v2) ใช้เฉพาะกรณี **ผ่านด่านสำเร็จ** เท่านั้น ไม่เกี่ยวกับกรณีแพ้ — แพ้กี่ครั้งก็ retry ด่านเดิมได้ไม่จำกัด
- แก้ที่ `GameOverScene.ts` (เพิ่มปุ่ม Retry, ส่ง `stageId` เดิมกลับไปที่ `GameClient.tsx` เพื่อ mount ใหม่) และตรวจ `/api/game/complete` ว่า record การแพ้ไม่ไปเขียนทับ `stagesCleared` ที่มีอยู่แล้วโดยผิดพลาด

## 5. ระบบโล่ (Shield) จากอุปกรณ์ — ฟีเจอร์ใหม่ (ไม่ใช่ HP, ไม่ฟื้นฟู)

- โล่เป็นหลอดพลังงานสีเทาแยกจากหลอด HP (แสดงซ้อน/ต่อกับหลอด HP ใน HUD เช่น หลอดเทาอยู่หน้าหลอดแดง) รับดาเมจก่อน HP เสมอ (ดาเมจตัดโล่ก่อน โล่หมดค่อยตัด HP)
- **โล่ไม่ฟื้นฟูเองระหว่างด่าน** เมื่อหมดแล้วหมดเลยจนกว่าจะเริ่มด่าน/ชีวิตใหม่ (reset เต็มทุกครั้งที่เข้าด่านใหม่หรือ retry)
- ค่าโล่มาจาก "อุปกรณ์ที่ equip อยู่" แบบบวกตรงๆ (แต้มดิบ ไม่ใช่ %) ตามตารางนี้ (เพิ่มเข้า sheet `Equipment` เป็น column `shieldValue`):

| Rarity | หมวก | เกราะ | รองเท้า |
|---|---|---|---|
| common | 75 | 100 | 50 |
| rare | 150 | 200 | 100 |
| epic | 300 | 400 | 200 |
| legendary | 600 | 800 | 400 |

- โล่รวม = ผลรวม shieldValue ของหมวก+เกราะ+รองเท้าที่ equip อยู่ (ไม่ต้องคูณ % จาก passive ใดๆ เป็นค่าคงที่ตรงๆ) คำนวณพร้อมกับ stat อื่นตอน `/api/game/start`

## 6. ตกแต่งชุดเกราะ/หมวก/รองเท้าใหม่ตามธีมวัสดุ (แทนที่ sprite เดิมทั้งหมด)

Sprite 12 ไฟล์ใหม่ (ข้อบนสุดของไฟล์นี้) ออกแบบตามธีมนี้แล้ว ให้ผูกเข้าโค้ดแทนของเดิมทั้งหมด:
- **Common** = หนังสัตว์ (สีน้ำตาลทำจากหนัง มีรอยเย็บ)
- **Rare** = เหล็ก (สีเทาโลหะ มีหมุดย้ำ)
- **Epic** = เพชรสีม่วง (ไล่เฉดม่วง มีเหลี่ยมเพชรตรงกลาง)
- **Legendary** = ทองคำอร่าม (ไล่เฉดทอง มีลวดลายดาว/rim เรืองแสง ดูยิ่งใหญ่)

อัปเดต `PreloadScene.ts` ให้โหลด sprite key ตาม `equipmentId` + `rarity` (เช่น `helmet_epic`, `vest_legendary`) แทนของเดิม

## 7. มินิแมพ (Mini-map) มุมซ้ายบนระหว่างเล่น — ฟีเจอร์ใหม่

- เพิ่ม `MiniMapScene.ts` (หรือ overlay UI ใน `HUDScene.ts`) แสดงมุมซ้ายบนของจอตลอดเวลาเล่น
- แสดง: ตำแหน่งผู้เล่น (จุดสีเขียว), ตำแหน่งศัตรูทั้งหมดที่ยังไม่ตาย (จุดสีแดง), ขอบเขตแมพย่อส่วน (เทียบสัดส่วนจาก stage width/height)
- อัปเดตตำแหน่งแบบ real-time ทุกเฟรม (หรือ throttle ทุก 100-150ms เพื่อ performance) โดยดึงพิกัดจริงจาก player/enemy sprites แล้วแปลงสัดส่วนเป็นพิกัดในกรอบมินิแมพ (เช่น กรอบขนาด 140x100px มุมซ้ายบน)
- ใช้ `Phaser.GameObjects.Graphics` วาดจุดแบบง่าย ไม่ต้องทำเป็น texture จริงของแมพ (แค่กรอบ + จุดสีพอ ประหยัด performance)

## 8. สิ่งกีดขวางเพิ่มเติม (ใช้ sprite ใหม่ที่สร้างไว้)

เพิ่มความหลากหลายของ cover object ในด่าน โดยใช้ sprite ใหม่: `obstacle_tree.svg` (ต้นไม้), `obstacle_wall.svg` (กำแพงคอนกรีต), `obstacle_house.svg` (บ้านจิ๋ว), `obstacle_camp_tent.svg` (เต็นท์แคมป์) ร่วมกับของเดิม `cover_sandbag.svg`, `cover_crate.svg`

- แก้ `CoverObject.ts` ให้รองรับหลาย sprite type (enum/type ตาม obstacle ที่ใช้) แต่ละแบบมี hitbox ขนาดต่างกันให้เหมาะสมกับภาพ (ต้นไม้/บ้าน hitbox ใหญ่กว่ากล่อง/กระสอบทราย)
- เพิ่มจำนวน cover object ต่อด่านให้มากขึ้นจากปัจจุบัน (สุ่มวางตำแหน่งจากรายการ obstacle ทั้ง 6 แบบผสมกัน ไม่ใช้แบบเดียวซ้ำทั้งด่าน) เพื่อให้แมพดูมีชีวิตชีวาและมีที่กำบังหลากหลายให้เล่นเชิงกลยุทธ์มากขึ้น
